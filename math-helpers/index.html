How to use in Funebra

In the Funebra UI, set:

  • Steps → number of points (e.g., 12*30)
  • X(o) → expression returning x
  • Y(o) → expression returning y
  • Z(o) → expression returning z (or 0)

Rule of thumb: Steps = edges × stepsPerEdge (stars use 2×sides edges).

// Declare once in your script
const TAU = Math.PI * 2;

// Example: 5-point star (2D)
Steps: 10*36
X(o): starX(o, 5, 150, 70, 360, 36)
Y(o): starY(o, 5, 150, 70, 260, 36)
Z(o): 0

polygonX / polygonY

Signature: polygonX(o, sides, radius, centerX, stepsPerEdge), polygonY(o, sides, radius, centerY, stepsPerEdge)

Description: Piecewise-linear interpolation around a regular polygon with sides edges and radius radius. Step index o advances along edges; stepsPerEdge controls resolution.

Parametric form (LaTeX)

Let \( e=\left\lfloor \dfrac{o}{s} \right\rfloor,\; t=\dfrac{o\bmod s}{s},\; \theta_1=\dfrac{2\pi e}{n},\; \theta_2=\dfrac{2\pi (e+1)}{n} \). Then

\( x = (1-t)(R\cos\theta_1+C_x) + t(R\cos\theta_2+C_x) \) \( y = (1-t)(R\sin\theta_1+C_y) + t(R\sin\theta_2+C_y) \)

Ready-to-paste (hexagon)

Steps: 6*40
X(o): polygonX(o, 6, 140, 360, 40)
Y(o): polygonY(o, 6, 140, 260, 40)
Z(o): 0

starX / starY

Signature: starX(o, points, R1, R2, center, stepsPerEdge), starY(o, points, R1, R2, center, stepsPerEdge)

Description: Alternates outer radius R1 and inner radius R2 around points tips (total edges = 2×points), interpolating linearly along edges.

Ready-to-paste (5-point star)

Steps: 10*36
X(o): starX(o, 5, 150, 70, 360, 36)
Y(o): starY(o, 5, 150, 70, 260, 36)
Z(o): 0

crossX / crossY / crossZ

Signature: crossX(o, stepsPerEdge, cx, cy, armW, armT, stemH, stemT, skew, rot), crossY(...), crossZ(o, steps, cx, cy, armW, armT, stemH, stemT, amp, phase, zCenter, pitch)

Description: Outline of a Latin cross (12 edges). Optional crossZ adds sinusoidal/helix depth.

heart2D_x / heart2D_y

Signature: heart2D_x(o, steps, scale, cx), heart2D_y(o, steps, scale, cy)

Formula: classic cardioid-like heart \( x = 16\sin^3 t,\; y = 13\cos t - 5\cos(2t) - 2\cos(3t) - \cos(4t) \), scaled and centered.

Steps: 720
X(o): heart2D_x(o, 720, 6, 360)
Y(o): heart2D_y(o, 720, 6, 260)
Z(o): 0

FAQ

What is a “Funebraic equation”?

A parametric or procedural expression of point movement law used by the Funebra engine. Instead of y = f(x), we provide X(o), Y(o), and optionally Z(o) to trace a curve or surface in discrete steps.

Can I reproduce these in Desmos or Google Sheets?

Yes. Most helpers have a piecewise parametric equivalent using floor and fractional parts. See our Desmos and Sheets examples (link them from here) for polygons and stars.
